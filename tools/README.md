# 工具使用指南

本文档旨在详细说明 `tools` 目录下各自动化工具的功能特性与使用方法。

## 1. Markdown 链接检查工具

本章节主要介绍项目中提供的链接有效性检查工具的功能特性及其具体使用方法。

### 1.1 工具功能简介

本小节将说明链接检查脚本的核心功能与设计考量。

`check_links.py` 是一个基于 Python 开发的自动化脚本，专门用于验证 Markdown 文档中超链接的有效性。该工具在设计上充分考虑了文件名中包含空格、URL 编码处理以及 Git 子模块未初始化等复杂场景，能够准确区分并分别校验本地文件路径与外部网络地址。每次执行后，系统将在项目根目录自动生成一份详细的检测报告（默认输出为 `link_check_report_v2.txt`）。

### 1.2 命令行参数说明

本小节将详细列出该工具支持的所有命令行参数及其具体含义。

该脚本通过 `argparse` 模块提供灵活的命令行交互能力，主要支持以下参数：

| 参数名称 | 选项范围                   | 默认值                       | 功能描述                                                                                          |
| -------- | -------------------------- | ---------------------------- | ------------------------------------------------------------------------------------------------- |
| `--all`  | 无                         | 仅检查项目根目录的 README.md | 是否全量检查项目内的所有 Markdown 文件（执行时将自动跳过隐藏目录及环境依赖目录）。                |
| `--type` | `local`, `external`, `all` | `all`                        | 指定需要检查的链接类型。其中 `local` 表示仅检查本地文件路径，`external` 表示仅检查外部 URL 地址。 |

### 1.3 典型使用示例

本小节将提供几个常见场景下的命令执行示例，以帮助用户快速上手。

以下代码块展示了如何通过终端运行该检查工具：

```bash
# 示例 1：默认行为，仅检查项目根目录下的 README.md 文件中的所有链接
python tools/check_links.py

# 示例 2：全量检查项目中所有的 Markdown 文件，排查所有本地与外部链接
python tools/check_links.py --all

# 示例 3：全量检查项目中所有的 Markdown 文件，但仅校验外部网络链接
python tools/check_links.py --all --type external

# 示例 4：仅检查 README.md 文件中的本地文件路径引用是否有效
python tools/check_links.py --type local
```
